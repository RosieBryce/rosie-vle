CNNs

Convolutional Networks — Key Concepts

CNNs exploit spatial structure through local receptive fields (each neuron sees only a patch), weight sharing (same filter applied everywhere), and pooling (spatial downsampling).

ConceptDetails
Conv2d output size (no padding)floor((H − k) / stride) + 1 per spatial dim
padding='same'Pads to preserve spatial dimensions (output same H × W as input)
padding='valid'No padding — output is smaller than input
Parameter count(k × k × C_in + 1) × C_out per conv layer
AdaptiveAvgPool2d(1)Reduces any spatial size to (B, C, 1, 1) — output is always 1×1 regardless of input size
Why AdaptiveAvgPool2d over Flatten for transfer learning: Flatten produces a vector whose size depends on input resolution — a parameter explosion. Global average pooling reduces each feature map to one scalar, giving a fixed-size output regardless of input size and far fewer parameters.
Sequence Models

RNN vs LSTM vs GRU — The Critical Differences

All three process sequences step-by-step, passing hidden state forward. They differ in how much of the past they can remember and at what computational cost.

RNNLSTMGRU
State vectors h only h + c (cell state) h only (combined gate)
Vanishing gradient Severe Mostly solved Mostly solved
Parameters Fewest Most Middle
Speed Fastest Slowest Faster than LSTM
GRU has no cell state — only hidden state. LSTM has both h_n and c_n. This is a common exam question. GRU achieves comparable performance with fewer parameters.
Transfer Learning

Transfer Learning in PyTorch — Model Anatomy

Each torchvision model has a different attribute name for its final classification layer. You must know the correct attribute to replace it:

ModelFinal layer attributeTypical use
ResNet-18/50model.fcGeneral purpose — good balance
VGG-16model.classifier[-1]Older benchmark; large parameter count
EfficientNetmodel.classifier[-1]Best accuracy-per-parameter
MobileNetmodel.classifier[-1]Mobile/edge deployment

After freezing layers (param.requires_grad = False), pass only trainable parameters to the optimiser using filter(lambda p: p.requires_grad, model.parameters()). Transfer learning benefits most when your dataset is small and similar to the pretraining domain.

Architecture

Bidirectional RNNs & LSTM Output Shapes

A bidirectional LSTM processes the sequence in both forward and backward directions, then concatenates the outputs. This doubles the output hidden size.

ConfigurationOutput shape (batch_first=True)
Unidirectional LSTM, hidden_size=256(batch, seq_len, 256)
Bidirectional LSTM, hidden_size=256(batch, seq_len, 512) — doubled
h_n from unidirectional(num_layers, batch, hidden_size)
h_n from bidirectional(num_layers × 2, batch, hidden_size)
Dropout in LSTM only applies between layers (not within a single layer), and is NOT applied to the last layer's output. If num_layers=1, setting dropout has no effect.

Exam Traps
⚠ Trap 1 — batch_first default
nn.LSTM default is (seq_len, batch, features) — not batch-first. If you feed (batch, seq_len, features) without batch_first=True, the LSTM silently treats your batch dimension as the sequence length. Training will appear to run but produce wrong results.
⚠ Trap 2 — LSTM returns a tuple
nn.LSTM returns (output, (h_n, c_n)). If you write out = self.lstm(x) and then do out[:, -1, :], you're indexing a tuple — this raises a TypeError or produces nonsense results. Always unpack: out, (h_n, c_n) = self.lstm(x).
⚠ Trap 3 — GRU has no cell state
GRU returns (output, h_n) — there is no c_n. LSTM returns (output, (h_n, c_n)). Mixing up these return signatures is a common source of unpacking errors. Remember: GRU is simpler — one state vector, not two.
⚠ Trap 4 — model.eval() does NOT disable gradients
model.eval() disables dropout and switches BatchNorm to inference mode. It does NOT disable gradient computation. For that you need torch.no_grad() separately. Always use both during evaluation: model.eval() + with torch.no_grad():.
⚠ Trap 5 — Bidirectional doubles output size
A bidirectional LSTM with hidden_size=256 produces output of size 512. The linear layer after it must accept 512 inputs, not 256. Forgetting this causes a shape mismatch error at the linear layer.

Common Gotchas
  • batch_first=True in LSTM — default is (seq_len, batch, features); most expect batch-first
  • LSTM returns a tuple — always unpack out, (h_n, c_n) = self.lstm(x)
  • Conv2d input must be 4D — (batch, channels, height, width); a single image needs unsqueeze(0)
  • ToTensor() changes channel order — HWC (PIL) → CHW (PyTorch); don't manually transpose
  • ImageNet normalisation only when using pretrained weights — use dataset mean/std otherwise
  • Forgetting .detach() when logging losses — accumulates computation graphs across epochs
CNN Pattern
import torch.nn as nn

class CNN(nn.Module):
    def __init__(self, num_classes):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),  # (B,3,H,W) → (B,32,H,W)
            nn.BatchNorm2d(32),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),                          # → (B,32,H/2,W/2)
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(inplace=True),
            nn.MaxPool2d(2, 2),                          # → (B,64,H/4,W/4)
        )
        self.classifier = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),                      # → (B,64,1,1)
            nn.Flatten(),                                 # → (B,64)
            nn.Linear(64, num_classes)
        )

    def forward(self, x):
        return self.classifier(self.features(x))
Transfer Learning Pattern
import torchvision.models as models

backbone = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V1)

# Freeze all layers
for param in backbone.parameters():
    param.requires_grad = False

# Replace final layer
backbone.fc = nn.Linear(backbone.fc.in_features, num_classes)

# Only trainable params to optimiser
optimizer = torch.optim.Adam(
    filter(lambda p: p.requires_grad, backbone.parameters()),
    lr=1e-3
)
LSTM Classifier Pattern
class LSTMClassifier(nn.Module):
    def __init__(self, input_size, hidden_size, num_layers, num_classes):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size, hidden_size, num_layers,
            batch_first=True,      # CRITICAL
            dropout=0.3 if num_layers > 1 else 0
        )
        self.fc = nn.Linear(hidden_size, num_classes)

    def forward(self, x):
        # x: (batch, seq_len, input_size)
        out, (h_n, c_n) = self.lstm(x)     # unpack tuple
        last = out[:, -1, :]               # last timestep
        return self.fc(last)
