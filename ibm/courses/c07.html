Generative AI and LLMs: Architecture and Data Preparation

The shortest course and the pivot point from discriminative deep learning to generative AI. Core concepts: what makes a model generative, tokenisation, pretraining objectives, and data preparation at scale.

Generative vs Discriminative

Generative vs Discriminative Models

DiscriminativeGenerative
Goal P(y|x) — classify input P(x) or P(x|y) — model the data distribution
Examples CNN classifier, logistic regression GPT, VAE, diffusion models
Can generate new data? No Yes

LLMs are generative because they learn a distribution over token sequences and can sample from it. Generating text = sampling from that distribution one token at a time.
Architecture

The LLM Family Tree

Encoder-Only
BERT, RoBERTa, DistilBERT
Bidirectional context. Best for understanding: classification, NER, embeddings. NOT generative.
Decoder-Only
GPT-2/3/4, LLaMA, Mistral
Autoregressive: predicts next token from previous. Dominant architecture for text generation.
Encoder-Decoder
T5, BART, Flan-T5
Encoder reads input, decoder generates output. Best for translation, summarisation, Q&A.
Tokenisation

Tokenisation — The Most Underestimated Concept

Tokenisation converts raw text to integer IDs. It is NOT simple word-splitting.

MethodUsed byKey property
BPE (Byte-Pair Encoding) GPT-2, GPT-3, LLaMA, Falcon Starts with characters, merges most frequent adjacent pairs iteratively
WordPiece BERT Similar to BPE; subwords prefixed with ## if not at word start
SentencePiece T5, LLaMA (some) Language-agnostic; works directly on Unicode
Key facts: 1 token ≈ ¾ word (English). Context window is measured in tokens. Different models have different vocabularies — tokens are NOT interchangeable. Tokenisation is not just space-splitting: "don't" → ["don", "'t"].
Pretraining

Pretraining Objectives — CLM vs MLM

ObjectiveArchitectureHow it works
Causal LM (CLM) Decoder-only (GPT) Predict next token given all previous tokens — autoregressive, unidirectional
Masked LM (MLM) Encoder-only (BERT) Predict ~15% masked tokens using full bidirectional context
Span corruption Encoder-decoder (T5) Mask spans of tokens with sentinels; decoder reconstructs them
CLM vs MLM explains the BERT/GPT capability split: GPT-style models cannot look at future tokens during training (causal masking) — good for generation. BERT looks in both directions — good for understanding but cannot autoregressively generate text.

Exam Traps
⚠ Trap 1 — Encoder-only models are NOT generative
BERT, RoBERTa, DistilBERT cannot autoregressively generate text. They are encoder-only — no mechanism for producing sequences token-by-token. For generation you need a decoder (GPT) or encoder-decoder (T5/BART).
⚠ Trap 2 — BPE starts with characters, not words
BPE does not split on spaces first. It begins with individual characters (or bytes) and iteratively merges the most frequent adjacent pair. Common words become single tokens; rare words are split into subword units. This is why "tokenisation" might become ["token", "isation"] in some models.
⚠ Trap 3 — Context window is in tokens, not words
The context window limit is measured in tokens, not words or characters. 1 English word ≈ 1.3 tokens. A 4096-token context window holds roughly 3000 words. Estimate token count with len(tokenizer(text)['input_ids']), not len(text.split()).
⚠ Trap 4 — Pretraining is self-supervised, not unsupervised
LLM pretraining is self-supervised — labels are derived automatically from the data (next token, masked token). This is more precise than calling it "unsupervised" — there ARE targets, they're just constructed from the input itself, requiring no human annotation.
⚠ Trap 5 — Fine-tuning updates weights; prompting does not
Fine-tuning modifies model parameters. Few-shot prompting, zero-shot prompting, and RAG do NOT update any weights. Temperature, top-k, and top-p are inference-time parameters — they also do not affect weights.

Common Gotchas
  • Tokens ≠ words — always use tokeniser to count tokens, not len(text.split()) for cost or context estimates
  • Tokenisers are not interchangeable — GPT-2 weights with LLaMA tokeniser = gibberish; each model's vocabulary is trained with its weights
  • Special tokens matter — BERT needs [CLS] and [SEP]; GPT uses <|endoftext|>; chat models need role tags. Missing them degrades performance
  • Context window ≠ memory — the model has no persistent memory between conversations; it "remembers" only what's in the current context window
  • Benchmark contamination — if test benchmarks appeared in pretraining data, scores are inflated. This is a real problem in LLM evaluation
Tokenisation Example
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained('gpt2')
text = "Hello, world! Tokenisation isn't just word-splitting."
tokens = tokenizer(text)
print(tokens['input_ids'])           # list of integer IDs
print(len(tokens['input_ids']))      # token count (more than word count)

# Decode back
for token_id in tokens['input_ids']:
    print(repr(tokenizer.decode([token_id])))  # see each piece
