Algorithm Selection

When to Use Which Algorithm

Algorithm Best for Needs scaling? Interpretable?
Linear Regression Continuous target, linear relationship Yes Yes
Ridge / Lasso Linear + regularisation needed Yes Yes
Logistic Regression Binary/multiclass classification Yes Yes
Decision Tree Non-linear, need interpretability No Yes
Random Forest General-purpose, handles missing No Partial
SVM (RBF kernel) Small-medium datasets, high dim Yes No
KNN Lazy baseline, no training cost Yes No
Key distinction: Tree-based methods (Decision Tree, Random Forest, Gradient Boosting) do NOT require feature scaling — they split on thresholds, not distances. Distance-based methods (KNN, SVM) and gradient-based methods (linear models, neural networks) require scaling.
Evaluation

Evaluation Metrics — Match to Task

MetricFormula / RangeUse when
Accuracy correct / total · [0,1] Classes are balanced
Precision TP / (TP + FP) False positives are costly (spam filter)
Recall TP / (TP + FN) False negatives are costly (cancer screening)
F1 Harmonic mean of P & R Imbalanced classes, both P & R matter
ROC-AUC [0.5, 1.0] — threshold-independent Ranking quality across all thresholds
R² 1 = perfect, 0 = mean baseline, can be negative Regression — proportion of variance explained

Regression also uses MAE (robust to outliers, same units as target) and RMSE (penalises large errors; in original units after sqrt).

High Bias (Underfitting)

→ Model too simple

High Variance (Overfitting)

Exam Traps
⚠ Trap 1 — Lasso vs Ridge zeros
Only L1 (Lasso) can drive weights exactly to zero. Ridge (L2) shrinks weights toward zero but never reaches it — Ridge cannot perform feature selection. Lasso is preferred when you suspect many features are irrelevant.
⚠ Trap 2 — Cross-validation does not train the final model
Cross-validation estimates generalisation error only. The final model is trained on all training data with the best hyperparameters found by CV. A common mistake: reporting CV fold accuracy as the final model's test accuracy.
⚠ Trap 3 — Accuracy on imbalanced classes
High accuracy on imbalanced data is meaningless. A 99% class → always-predict-negative → 99% accuracy, 0% recall for the minority class. Always check the confusion matrix; report F1 or per-class recall when classes are imbalanced.
⚠ Trap 4 — fit_transform on test data
Never call fit_transform() on the test set. The scaler's mean/std must come from the training set only. Fitting on test data leaks test statistics into the transformation, making test performance unrealistically optimistic.
⚠ Trap 5 — t-SNE is stochastic and non-deterministic
t-SNE produces different plots every run (unless you fix the random seed). It is also not suitable for ML features — it cannot transform new data points without re-running. Use UMAP if you need a dimensionality reduction for downstream ML.

Common Gotchas
  • Data leakage: always split first, then fit scalers/encoders on train only
  • KNN requires scaling — Euclidean distance is meaningless across different-scale features
  • SVM with RBF is O(n³) — avoid on large datasets; use linear SVM or other scalable models
  • Decision trees overfit by default — always set max_depth or min_samples_leaf
  • PCA must be fitted on train only, then applied to test — same as any other transformer
  • k=1 in KNN = zero training error (memorises training set), very high variance
  • Logistic Regression outputs probabilities — use predict_proba() when you need confidence scores
Full Pipeline Pattern
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import GridSearchCV, StratifiedKFold

pipe = Pipeline([
    ('scaler', StandardScaler()),
    ('clf',    LogisticRegression(C=1.0, max_iter=1000))
])

cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
param_grid = {'clf__C': [0.01, 0.1, 1, 10]}
gs = GridSearchCV(pipe, param_grid, cv=cv, scoring='f1_macro')
gs.fit(X_train, y_train)

print(gs.best_params_)
print(gs.score(X_test, y_test))
Cross-Validation Variants
from sklearn.model_selection import cross_val_score, KFold, StratifiedKFold

# For classification — always stratified
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring='f1_macro')
print(f"CV F1: {scores.mean():.3f} ± {scores.std():.3f}")

# For regression — plain KFold is fine
cv = KFold(n_splits=5, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=cv, scoring='r2')
