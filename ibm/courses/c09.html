Generation

Autoregressive Generation & Causal Masking

Decoder-only models (GPT family) generate text one token at a time. Each new token is conditioned on all previous:

P(t₁, t₂, ..., tₙ) = P(t₁) × P(t₂|t₁) × P(t₃|t₁,t₂) × ...

Causal masking enforces this — token i cannot attend to token j > i (future tokens). This upper-triangular mask enables parallel training (all positions computed at once) while maintaining the autoregressive property.

Each forward pass produces a probability distribution over the entire vocabulary. A decoding strategy decides which token to emit. This choice has a huge effect on the quality and character of generated text.
Decoding Strategies

Decoding Strategies — Know Each One Cold

StrategyHow it worksDeterministic?Use when
Greedy Always pick highest-probability token Yes Speed; simple structured output
Beam search Keep top-k sequences at each step Yes Translation, summarisation (conservative)
Sampling Sample randomly from full distribution No Creative tasks; diverse outputs needed
Top-k Sample from top k tokens only No Reduce incoherence while adding diversity
Top-p (nucleus) Sample from smallest set with cumulative prob ≥ p No General generation — preferred over top-k

Beam size=1 = greedy. Top-p=1.0 = sample from full vocabulary (no restriction). Top-p=0.0 ≈ greedy. Typical production settings: temperature=0.7, top_p=0.9.

Temperature

Temperature Scaling — The Creativity Dial

Temperature T divides the raw logits before softmax: probs = softmax(logits / T)

T → 0 (very small)
Peaked distribution. Approaches greedy (argmax). Deterministic, safe, boring.
T = 1.0
Unchanged distribution. Model's raw probabilities.
T > 1.0 (high)
Flatter distribution. More random, more creative, more risk of incoherence.
Temperature=0 causes division by zero — most libraries handle this by switching to greedy (argmax) when temperature is 0 or very small. Also: do_sample=False overrides temperature and top_p entirely — you always get greedy/beam regardless of other settings.
Evaluation & Efficiency

Perplexity & KV Cache

ConceptWhat it means
Perplexity exp(−1/N × Σ log P(token|context)) — how "surprised" the model is; lower = better language model
Perplexity = 10 Model is as uncertain as if choosing uniformly among 10 options
KV cache Caches key/value pairs from previous tokens to avoid recomputation at each generation step
KV cache effect Makes inference memory-bound, not compute-bound for long sequences; speeds up generation significantly
Perplexity is tokeniser-dependent — you cannot compare perplexity scores across models with different vocabularies. Models with larger vocabularies tend to have lower perplexity. Perplexity measures pretraining quality, not task performance.

Exam Traps
⚠ Trap 1 — Greedy is not globally optimal
Greedy search does NOT find the globally most probable sequence. It is locally optimal at each step. A sequence that requires a lower-probability intermediate token to reach a high-probability continuation will be missed entirely by greedy decoding. Beam search addresses this by maintaining multiple hypotheses.
⚠ Trap 2 — do_sample=False overrides temperature
If do_sample=False, temperature and top_p have no effect. With sampling disabled, you always get greedy decoding (or beam search if num_beams > 1). Temperature scaling only applies when sampling is enabled. Setting temperature without enabling sampling is a common silent mistake.
⚠ Trap 3 — max_length includes prompt tokens
max_length counts the prompt tokens too. If your prompt is 50 tokens and you set max_length=100, you only get 50 new tokens. Use max_new_tokens instead to specify how many new tokens to generate.
⚠ Trap 4 — Beam search is deterministic
Beam search is deterministic — same input, same output every time. Sampling-based methods (top-k, top-p, temperature) are stochastic — different output each run. KV cache does NOT affect model quality — it only affects inference speed.
⚠ Trap 5 — Perplexity cannot be compared across tokenisers
Perplexity scores are only meaningful when compared within the same tokeniser. A model with a larger vocabulary can artificially achieve lower perplexity because it assigns higher probability to each token. Comparing perplexity between GPT-2 and LLaMA is meaningless.

Common Gotchas
  • temperature=0 is division by zero — libraries handle it by switching to greedy; use do_sample=False explicitly instead
  • do_sample=False overrides temperature/top_p — if sampling is off, these parameters have no effect
  • max_length includes the prompt — use max_new_tokens to control output length independently
  • GPT-2 has no pad token — set pad_token_id=tokenizer.eos_token_id to avoid warnings
  • top_p and top_k can be combined — apply top_k first, then top_p, then sample
  • Beam search num_beams=1 = greedy — explicitly specifying num_beams=1 disables beam search
Text Generation Pattern
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

tokenizer = AutoTokenizer.from_pretrained('gpt2')
model     = AutoModelForCausalLM.from_pretrained('gpt2')

inputs = tokenizer("The future of AI is", return_tensors='pt')

outputs = model.generate(
    **inputs,
    max_new_tokens=100,         # not max_length — excludes prompt
    do_sample=True,             # enable sampling
    temperature=0.7,            # sharpen distribution slightly
    top_p=0.9,                  # nucleus sampling
    repetition_penalty=1.1,     # slight repetition penalty
    pad_token_id=tokenizer.eos_token_id
)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))
