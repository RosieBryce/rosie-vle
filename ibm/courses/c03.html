Execution Model

TF2 Eager vs Graph Execution

TF2 runs in eager mode by default — operations execute immediately, like NumPy. This makes debugging straightforward: you can inspect tensors, use Python control flow, and step through code normally.

@tf.function converts a Python function into a static computation graph, compiled once and reused. This is faster for production because TF can optimise the graph — but it breaks Python-side effects (print statements, list appends) that should run every call, because they only execute on the first trace.

Key distinction: Eager = Pythonic, debuggable, slower at scale. @tf.function = compiled graph, fast, but traces once — don't rely on Python side effects inside it.
Architecture

Functional API vs Sequential — When Each Applies

Situation Use Why
Simple linear stack, one input, one output Sequential Clean, minimal code
Multiple inputs (e.g. image + text combined) Functional Sequential has exactly one input tensor
Multiple outputs (e.g. classification + bounding box) Functional Sequential has exactly one output tensor
Skip connections (ResNet-style) Functional Layer outputs are just tensors — can be added, concatenated freely
Shared layers (same layer used twice) Functional Call the same layer object on different inputs

In the Functional API, each layer is called like a function: x = Dense(256)(inp). The result is a tensor. The model is defined by which tensors are inputs and outputs.

Transfer Learning

The Three-Stage Transfer Learning Pattern

Transfer learning reuses a model trained on a large dataset (usually ImageNet) as a feature extractor for a new task. The pretrained weights encode rich, general visual features; only the final classification head needs learning from scratch.

Stage 1
Load & Freeze
Load pretrained base with include_top=False. Set base.trainable = False. Attach new head.
Stage 2
Train Head Only
Compile and train with normal LR. Only the new Dense layers learn. Base weights unchanged.
Stage 3 (optional)
Fine-tune
Unfreeze top N layers. Recompile with very small LR (1e-5). Avoid destroying pretrained features.
Critical: When calling a frozen base model, always pass training=False explicitly: x = base(inputs, training=False). Without this, BatchNorm layers inside the base update their running statistics on your (small) dataset, corrupting the pretrained statistics.
Data Pipeline

tf.data: Why It Exists

Passing NumPy arrays to model.fit() loads everything into memory at once. tf.data is a streaming pipeline that feeds data to the GPU in parallel with training.

Capabilitytf.data approach
Parallelism.prefetch(AUTOTUNE) — CPU loads next batch while GPU trains on current one
Shuffling.shuffle(buffer_size) — randomises order without loading full dataset
Batching.batch(32) — always required before model.fit()
Large datasetsReads from files (images, TFRecords) without loading all into RAM

The canonical pipeline order: .shuffle() → .batch() → .prefetch(). Shuffle before batch (otherwise you shuffle entire batches, not individual examples).

Regularisation

BatchNorm: Training vs Inference Behaviour

BatchNorm normalises each feature across the batch. It behaves differently at training vs inference:

ModeNormalisation usesUpdates running stats?
training=True Current batch mean/variance Yes — accumulates running mean/var
training=False Stored running mean/variance No

BatchNorm has 4 parameters per feature: γ (scale) and β (shift) are learned; running mean and running variance are tracked statistics (not gradient-updated).

Module 3 — Transformers

Building Transformer Blocks in Keras

Keras provides tf.keras.layers.MultiHeadAttention as a built-in layer. You compose it with residual connections and LayerNorm to build a transformer encoder block.

ConceptIn KerasKey detail
Multi-head attentionMultiHeadAttention(num_heads, key_dim)key_dim is per head — total = num_heads × key_dim
Self-attentionmha(query=x, value=x)Same tensor as query and value; key defaults to value
Residual + NormLayerNormalization()(x + attn_output)Written explicitly in custom blocks
Positional encodingCustom layer (sinusoidal or learned)Added to token embeddings — not concatenated
Assessment trap: key_dim in Keras ≠ model dimension. A model with embed_dim=512 and num_heads=8 should use key_dim=64 (512/8).
Module 4 — Generative Models

Autoencoders, VAEs, and GANs

ModelWhat the encoder outputsCan generate new samples?Key mechanism
Autoencoder Deterministic latent vector z Poorly — latent space is unstructured Reconstruction loss (MSE or BCE)
VAE Distribution: mean μ + log σ² Yes — sample z ~ N(0,1), decode Reparameterisation trick + KL divergence
GAN Generator takes noise z directly Yes — G(z) → new sample Adversarial: G vs D training loop
Reparameterisation trick (VAE): instead of sampling z ~ N(μ, σ²) directly (not differentiable), compute z = μ + ε·σ where ε ~ N(0,1). Now gradients can flow through μ and σ.
GAN failure modes: mode collapse (generator produces limited variety) and training instability (G and D must stay balanced). Evaluated with FID score — lower is better.
Module 6 — Reinforcement Learning

Deep Q-Networks (DQN) with Keras

RL trains an agent to maximise cumulative reward by interacting with an environment. DQN uses a neural network to approximate Q(s, a) — the expected future reward for taking action a in state s.

RL termDefinition
State (s)Current observation of the environment
Action (a)Choice made by the agent
Reward (r)Scalar feedback from environment after taking action
Policy (π)Mapping from states to actions (what the agent learns)
Q-value Q(s,a)Expected cumulative reward from state s, taking action a, then following π
DQN componentWhy it's needed
Target network (frozen copy of Q-net)Training targets shift every update without it → oscillation and divergence
Experience replay bufferConsecutive frames are correlated; random mini-batch sampling gives i.i.d.-like training
Epsilon-greedy decayStart high ε (explore) → decay toward small ε (exploit) as agent learns

Exam Traps
⚠ Trap 1 — include_top
The assessment may ask what include_top=False does. The correct answer: it removes the final Dense classification head (the ImageNet softmax layer), leaving only the convolutional feature extraction layers. It does NOT disable any weights or freeze anything.
⚠ Trap 2 — training=False on frozen base
Why must you pass training=False when calling a frozen base model? BatchNorm layers in the base will update their running statistics even when trainable=False unless you explicitly pass training=False. Corruption of pretrained BatchNorm stats degrades performance — the base loses its ImageNet calibration.
⚠ Trap 3 — recompile after changing trainable
Changing layer.trainable has no effect until you call model.compile() again. The optimiser's list of parameters to update is frozen at compile time. Setting a layer's trainable attribute without recompiling means the optimiser still updates (or still skips) the old set of parameters.
⚠ Trap 4 — fine-tuning learning rate
Fine-tuning requires a much smaller learning rate than training from scratch — typically 10–100× smaller (e.g. 1e-5 instead of 1e-3). A high LR during fine-tuning destroys the pretrained feature representations (catastrophic forgetting), which defeats the entire point of using a pretrained model.
⚠ Trap 5 — @tf.function and Python side effects
@tf.function traces the function once and caches the compiled graph. Python-side effects inside the function (print statements, list appends, counters) only execute during the first trace. If you rely on them running every call, remove @tf.function or move the side effects outside.
⚠ Trap 6 — key_dim in MultiHeadAttention
key_dim is the dimension per attention head, not the total model dimension. A transformer with embed_dim=512 and num_heads=8 uses key_dim=64 (= 512 ÷ 8). The total attention output is num_heads × key_dim = 512, matching the embedding dimension. Confusing key_dim with the full model dimension is a common error.
⚠ Trap 7 — Autoencoder vs VAE generation
A regular autoencoder cannot reliably generate new samples. Its latent space is unstructured — random points in latent space decode to garbage. A VAE forces the latent space toward N(0,1) via the KL divergence term, so sampling z ~ N(0,1) and decoding actually produces valid outputs. The reparameterisation trick is what makes the VAE's sampling step differentiable.
⚠ Trap 8 — DQN target network purpose
The target network is not for evaluation — it's for training stability. Without a separate frozen target network, the Q-values you're training toward change at every update step (since both the Q-network and the targets depend on the same weights). This chasing of moving targets causes oscillation. The target network is updated every N steps, providing stable targets during training.

Common Gotchas
  • base(inputs, training=False) when base is frozen — must be explicit; model.fit() won't infer this automatically
  • Forgetting to recompile after changing trainable — the optimiser parameter list is set at compile time
  • GlobalAveragePooling2D vs Flatten — Flatten creates a huge parameter count; GAP reduces each feature map to one number. Always use GAP for transfer learning
  • Preprocessing mismatch — each tf.keras.applications model expects its own preprocessing: MobileNetV2 wants inputs in [-1,1], VGG16 wants BGR with ImageNet mean subtracted. Use the accompanying preprocess_input function
  • EarlyStopping without restore_best_weights=True — you'll get the last epoch weights, not the best epoch
  • Dataset not batched before model.fit() — always .batch() your tf.data pipeline
  • Forgetting .prefetch(AUTOTUNE) — GPU idles waiting for CPU to prepare the next batch
  • Conv2D output size confusion — padding='valid' reduces spatial dims; padding='same' preserves them
tf.data Pipeline Pattern
dataset = tf.data.Dataset.from_tensor_slices((X_train, y_train))
dataset = (dataset
    .shuffle(buffer_size=10_000)   # shuffle BEFORE batch
    .batch(32)
    .prefetch(tf.data.AUTOTUNE)    # overlap CPU prep with GPU training
)
model.fit(dataset, epochs=10)
Transfer Learning Pattern
base = tf.keras.applications.MobileNetV2(
    input_shape=(224, 224, 3),
    include_top=False,          # drop classification head
    weights='imagenet'
)
base.trainable = False          # freeze pretrained weights

inputs  = Input(shape=(224, 224, 3))
x       = base(inputs, training=False)  # CRITICAL: keeps BatchNorm frozen
x       = GlobalAveragePooling2D()(x)
x       = Dropout(0.2)(x)
outputs = Dense(n_classes, activation='softmax')(x)
model   = Model(inputs, outputs)

model.compile(optimizer='adam', loss='sparse_categorical_crossentropy')
model.fit(dataset, epochs=5)

# Fine-tuning (Stage 3):
base.trainable = True
for layer in base.layers[:-20]:
    layer.trainable = False

model.compile(optimizer=Adam(1e-5), ...)  # much smaller LR
model.fit(dataset, epochs=5)
Custom Training Loop
optimizer = tf.keras.optimizers.Adam(1e-3)
loss_fn   = tf.keras.losses.SparseCategoricalCrossentropy()

for epoch in range(epochs):
    for X_batch, y_batch in dataset:
        with tf.GradientTape() as tape:
            logits = model(X_batch, training=True)
            loss   = loss_fn(y_batch, logits)
        grads = tape.gradient(loss, model.trainable_variables)
        optimizer.apply_gradients(zip(grads, model.trainable_variables))

Use when you need control that model.fit() doesn't provide: custom metrics, gradient clipping, multiple losses, research experiments.

Key Architecture Patterns to Recognise
VGG-style:  [Conv-Conv-Pool] × N → Flatten/GAP → Dense → output
ResNet:     [Conv-BN-ReLU + skip connection] × N → output
Inception:  parallel conv branches (different kernel sizes) → concat → output
MobileNet:  depthwise separable convolutions (much cheaper than standard Conv)
