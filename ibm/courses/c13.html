Architecture

The QA Bot Architecture

The capstone builds a document question-answering bot. The required components:

1
Document loading — PDFs, text files, or web pages via LangChain loaders
2
Chunking and embedding — RecursiveCharacterTextSplitter → HuggingFace embeddings → FAISS
3
RAG chain — retriever + prompt template + LLM + output parser
4
Gradio interface — ChatInterface or Blocks with question input, answer output, source display
5
Source attribution — return_source_documents=True to display which chunks were used
6
Evaluation — test on known questions; RAGAS metrics; document results
Evaluation

RAG Evaluation Metrics (RAGAS)

RAG is hard to evaluate automatically. RAGAS provides three key automated metrics:

MetricWhat it measuresIdeal value
Faithfulness Does each claim in the answer appear in the source documents? (No hallucination) Close to 1.0
Answer Relevance Does the answer actually address the question? Close to 1.0
Context Recall Were the retrieved chunks actually relevant to the question? Close to 1.0
Even without RAGAS, demonstrate evaluation by manually testing at least 5 known questions and documenting whether the answers are correct and faithful to the documents.

Exam Traps
⚠ Trap 1 — RAG does not update model weights
This is the single most important conceptual distinction in the Gen AI block. RAG retrieves context and injects it at inference time. The model's weights never change. Fine-tuning updates weights. Prompting and RAG do not.
⚠ Trap 2 — Same embedding model required
The embedding model used during indexing must be identical to the one used during retrieval. This also means: if you rebuild the index with a different model, you must embed the query with the same new model. Mismatched embedding models produce meaningless similarity scores.
⚠ Trap 3 — share=True is temporary, not permanent
Gradio's share=True creates a tunnel URL that expires after 72 hours. It is not a deployment. For a permanent, publicly accessible URL, use Hugging Face Spaces or a cloud platform.
⚠ Trap 4 — ConversationalRetrievalChain vs RetrievalQA
RetrievalQA is single-turn (no memory). Each question is answered independently, with no awareness of previous conversation. ConversationalRetrievalChain handles multi-turn conversation using memory. For a chat bot, you need ConversationalRetrievalChain.
⚠ Trap 5 — Hallucination can occur even with retrieved context
A sufficiently capable LLM may prefer its parametric knowledge over the provided context. RAG reduces hallucination but does not eliminate it. Use explicit prompt instructions: "Answer ONLY using the provided context. If the answer is not in the context, say 'I don't know'." Even then, strong LLMs sometimes ignore this instruction.

Common Gotchas
  • Context window overflow — k=10 × chunk_size=500 chars ≈ 5000 tokens; add prompt overhead; may exceed model limit
  • LLM answers from memory not context — add explicit "Answer ONLY from context; if not in context, say I don't know"
  • FAISS index not rebuilt when docs added — use vectorstore.add_documents(new_chunks) to extend existing index
  • Gradio global state in multi-user app — use gr.State() for per-session vectorstore; global = users overwrite each other
  • HuggingFaceHub rate limits — free tier is strict; cache results or use local model for demos
  • PyPDFLoader fails on scanned PDFs — use UnstructuredPDFLoader or an OCR-based approach
Full Application Pattern
import gradio as gr
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.vectorstores import FAISS
from langchain.chains import RetrievalQA
from langchain.prompts import PromptTemplate

def build_qa_chain(pdf_path):
    loader   = PyPDFLoader(pdf_path)
    docs     = loader.load()
    splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
    chunks   = splitter.split_documents(docs)
    embedder = HuggingFaceEmbeddings(model_name='all-MiniLM-L6-v2')
    vs       = FAISS.from_documents(chunks, embedder)

    prompt = PromptTemplate(
        template="Context: {context}\nQuestion: {question}\n"
                 "Answer ONLY from the context. If not in context, say 'I don't know'.\nAnswer:",
        input_variables=['context', 'question']
    )
    return RetrievalQA.from_chain_type(
        llm=llm,
        retriever=vs.as_retriever(search_kwargs={'k': 3}),
        chain_type_kwargs={'prompt': prompt},
        return_source_documents=True
    )

def respond(question, history, chain):
    result  = chain({'query': question})
    answer  = result['result']
    sources = '\n'.join(set(d.metadata.get('source', '') for d in result['source_documents']))
    return f"{answer}\n\nSources: {sources}"

with gr.Blocks() as demo:
    chain_state = gr.State()
    file_input  = gr.File(label="Upload PDF")
    load_btn    = gr.Button("Load")
    chatbot     = gr.Chatbot()
    question    = gr.Textbox(placeholder="Ask a question...")
    submit_btn  = gr.Button("Ask")

    load_btn.click(build_qa_chain, inputs=file_input, outputs=chain_state)
    submit_btn.click(respond, inputs=[question, chatbot, chain_state], outputs=chatbot)

demo.launch()
