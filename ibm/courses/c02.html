Activation Functions

Activation Functions — Which to Use Where

FunctionRangeUse caseGotcha
ReLU — max(0,x) [0, ∞) Hidden layers (default choice) Dying ReLU: neurons can become permanently zero
Leaky ReLU (−∞, ∞) When ReLU neurons die Small negative slope (0.01x) keeps neurons alive
Sigmoid — 1/(1+e⁻ˣ) (0, 1) — never 0 or 1 Binary output layer only Vanishing gradient — saturates for large |x|
Softmax Probabilities summing to 1 Multi-class output layer only Never use in hidden layers
Linear / None (−∞, ∞) Regression output No squashing — appropriate for unbounded targets
Key insight: Without any activation function, stacking layers is equivalent to a single linear transformation. Activations are what give neural networks their ability to learn non-linear decision boundaries.
Loss Functions

Loss Function — Activation Matching

TaskOutput activationLoss functionLabel format
Binary classification Sigmoid binary_crossentropy 0 or 1
Multi-class (one-hot labels) Softmax categorical_crossentropy [0,0,1,0,...]
Multi-class (integer labels) Softmax sparse_categorical_crossentropy 2, 5, 0...
Regression Linear (none) mse or mae Float values

sparse_categorical_crossentropy is simply more convenient — it accepts integer labels directly and uses less memory. Functionally identical to one-hot + categorical_crossentropy.

Gradient Flow

Vanishing & Exploding Gradients

Backpropagation multiplies gradients through layers via the chain rule. In deep networks this causes two failure modes:

ProblemSymptomFix
Vanishing gradients Early layers don't learn; loss plateaus ReLU activations, BatchNorm, residual connections
Exploding gradients NaN loss, huge weight updates Gradient clipping (clipnorm=1.0), weight initialisation
Why sigmoid causes vanishing gradients: The sigmoid derivative peaks at 0.25 and approaches 0 for large |x|. Multiply 0.25 through 20 layers: 0.25²⁰ ≈ 9×10⁻¹³. Gradients effectively die before reaching early layers.
Optimisation

Gradient Descent Variants & Optimisers

VariantBatch sizeProsCons
Batch GD Full dataset Stable gradient estimate Slow per step, needs all data in memory
Stochastic GD (SGD) 1 sample Fast, escapes local minima Very noisy; unstable convergence
Mini-batch GD 32–256 Balance of both; GPU-efficient Batch size is a hyperparameter

Adam is the default optimiser for most problems — it combines RMSProp (adaptive per-parameter LR) and momentum. It adapts the learning rate per parameter throughout training — it is NOT a fixed learning rate.

AdaGrad accumulates all past gradients — learning rate shrinks to near-zero over long training. Generally not recommended for deep learning.

Exam Traps
⚠ Trap 1 — Dropout is training-only
Dropout is active only during training, not inference. Keras automatically disables it when you call model.predict() or model.evaluate(). If calling the model manually, you must pass training=False. Evaluating with dropout active randomly degrades scores.
⚠ Trap 2 — Sigmoid range is open interval
Sigmoid output is in (0, 1) — it never actually reaches 0 or 1. This matters for numerical stability (log(0) is undefined) and is why the combined BCEWithLogitsLoss / binary_crossentropy is numerically more stable than computing sigmoid separately then log.
⚠ Trap 3 — More layers is not always better
Stacking more layers brings diminishing returns and increases the vanishing gradient problem. Without residual connections, very deep networks often train worse than shallower ones. Depth is not a substitute for the right architecture or sufficient data.
⚠ Trap 4 — Adam adapts per parameter
Adam is NOT a fixed learning rate throughout training. It maintains separate exponentially decaying estimates of gradient (momentum) and squared gradient (RMSProp) per parameter. The effective step size adapts based on each parameter's history.
⚠ Trap 5 — Weight initialisation matters
Keras sets correct defaults — He init for ReLU, Glorot/Xavier for tanh/sigmoid. If you initialise all weights to zero, all neurons compute the same output and the same gradient — they never differentiate. This is called the symmetry problem and prevents learning entirely.

Common Gotchas
  • Forgetting to normalise inputs — different-scale features cause some weights to dominate; always scale to [0,1] or z-score
  • Wrong output activation — Softmax for multi-class, Sigmoid for binary, Linear for regression; using Softmax for regression is a classic error
  • Loss/activation mismatch — Sigmoid output + categorical_crossentropy is incorrect; match them always
  • Learning rate too high — loss oscillates or goes NaN; start at 1e-3 (Adam default), reduce if unstable
  • Not plotting validation — you need validation_split or validation_data to diagnose overfitting
  • Epoch ≠ step — one epoch = one full pass over training data; steps per epoch = len(X_train) / batch_size
Keras Sequential Pattern
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense, Dropout, BatchNormalization

model = Sequential([
    Dense(128, activation='relu', input_shape=(n_features,)),
    BatchNormalization(),
    Dropout(0.3),
    Dense(64, activation='relu'),
    Dense(n_classes, activation='softmax')   # multi-class output
])

model.compile(
    optimizer='adam',
    loss='sparse_categorical_crossentropy',   # integer labels
    metrics=['accuracy']
)

history = model.fit(
    X_train, y_train,
    epochs=50,
    batch_size=32,
    validation_split=0.2
)
Reading Training Curves
import matplotlib.pyplot as plt

# Good: both curves go down together
# Overfitting: val_loss increases while train_loss decreases
# Underfitting: both stay high (plateau)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].plot(history.history['loss'],     label='train')
axes[0].plot(history.history['val_loss'], label='val')
axes[0].set_title('Loss')
axes[0].legend()

axes[1].plot(history.history['accuracy'],     label='train')
axes[1].plot(history.history['val_accuracy'], label='val')
axes[1].set_title('Accuracy')
axes[1].legend()
